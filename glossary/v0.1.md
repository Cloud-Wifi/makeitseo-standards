## Abstract

This document is the canonical glossary for the MakeitSEO Standards Programme. It collects terms used across the constituent specifications, provides authoritative definitions, and notes where each term is principally introduced.

When the constituent specs differ on terminology, this glossary is authoritative. Editorial drift between specs should be reconciled toward this document; conflicts surfaced via GitHub Issues are resolved by updating either the offending spec or, where appropriate, this glossary.

---

## Status

This is the v0.1 draft of the cross-spec glossary. It is intentionally minimal — covering the terms most frequently used across multiple specs without aspiring to be a comprehensive technical dictionary. v0.2 may expand the term set; v1.0 may introduce sub-glossaries per domain (SEO terms, AEO terms, GEO terms, ARC terms).

## Terms

### Acknowledgement

The operator's marking of a cascade event as reviewed. Acknowledged events remain in the database for forensics but are removed from active alert queues. See `cascade-monitoring/v0.1`.

### AEO — Answer Engine Optimisation

The methodology pillar measuring inclusion in deterministic answer surfaces (AI Overviews, featured snippets, knowledge panels, People Also Ask). One of three outcome pillars. See `aeo-scoring/v0.1`.

### Affinity score

A customer's per-platform citation rate within a measurement window. Computed as `citation_count / query_count`. See `per-platform-source-affinity/v0.1`.

### Agent Runtime Readiness

A derived cross-pillar diagnostic view summarising runtime compatibility. Computed across the three pillars' runtime-compatibility sub-scores. Reported alongside the three pillar scores. Not itself a pillar score. See `agent-runtime-compatibility/v0.1`.

### AI citation

A measured event where a customer's domain appears as a cited source in an AI engine's response to a tracked query. See `geo-scoring/v0.1`.

### AI crawler

A web crawler operated by an AI engine vendor (or its delegated infrastructure provider) for the purpose of fetching site content into the AI engine's retrieval index, training set, or live-retrieval pipeline. See `ai-crawler-accessibility/v0.1`.

### Answer engine

A search-engine or AI-engine surface that produces a direct answer rather than (or in addition to) a ranked list of links. As of May 2026 the principal deterministic answer surfaces are Google AI Overviews, featured snippets, knowledge panels, People Also Ask. See `aeo-scoring/v0.1`.

### Applicability

The property that a given compatibility check is meaningful for a given runtime. Not every check applies to every runtime; the applicability matrix documents which check × runtime pairs are meaningful. See `agent-runtime-compatibility/v0.1`.

### ARC — Agent Runtime Compatibility

The methodology spec defining how agent runtimes are taxonomised, characterised, and measured. ARC is a measurement-vocabulary spec, not a pillar spec; it is referenced by all three outcome pillar specs for their runtime sub-scores. See `agent-runtime-compatibility/v0.1`.

### Brand variant

A registered alternative form of a customer's brand name (e.g. abbreviations, alternative spellings, trade names). See `brand-web-mentions/v0.1`.

### Brand web mention

Any mention of a customer's brand name (canonical form or registered variant) on the open web, with or without a clickable link. See `brand-web-mentions/v0.1`.

### Cascade event

An emitted record indicating that a measured signal has changed beyond a threshold within a measurement window. See `cascade-monitoring/v0.1`.

### Cascade weighting

The methodological question (deferred for v0.2 review per `methodology-adr/v0.1` Deferred 2) of whether ARC runtime failures should additionally depress citation outcomes in the GEO pillar beyond the runtime sub-score effect. See `cascade-monitoring/v0.1`.

### Characteristic

A specific behavioural property of a runtime that affects compatibility with websites (e.g. `js_execution`, `session_model`, `relevant_outcome_pillar`). See `agent-runtime-compatibility/v0.1`.

### Citation

See AI citation.

### Citation rate

The rate at which the customer's domain appears as a cited source across tracked AI engine responses. The principal outcome measurement in the GEO pillar. See `geo-scoring/v0.1`.

### Cohort

A grouping of customers sharing relevant traits for analysis purposes (e.g. `csr_heavy`, `ssr_clean`, `mixed`, `crawler_blocked`, `fully_open`). Used for the v0.2 cascade-weighting review. See `cascade-monitoring/v0.1`.

### Compatibility finding

The result of evaluating a single (check, runtime) pair against a website. One of: `compatible`, `incompatible`, `not-applicable`, `indeterminate`. See `agent-runtime-compatibility/v0.1`.

### Content age

The interval, in days, between the most recent date signal on a page (`dateModified`, `datePublished`, or HTTP `Last-Modified`) and the time of measurement. See `content-freshness/v0.1`.

### Default indeterminate

A verdict in AI crawler accessibility indicating that robots.txt exists but does not address the specific crawler. See `ai-crawler-accessibility/v0.1`.

### Deterministic answer surface

A search-result feature that displays a structured answer or entity summary derived from a specific page or set of pages via a deterministic selection process. Featured snippets, AI Overviews (the citation aspect), knowledge panels, PAA boxes. See `aeo-scoring/v0.1`.

### E-E-A-T

Experience, Expertise, Authoritativeness, Trustworthiness — Google's framework for evaluating content quality and source credibility. Not a single score; a bundle of signals. See `seo-scoring/v0.1`.

### Event type

A classification of the kind of change detected by cascade monitoring. v0.1 defines four: `brand_mention_spike`, `freshness_drop`, `affinity_shift`, `schema_gap`. See `cascade-monitoring/v0.1`.

### Explicitly allowed

A verdict in AI crawler accessibility indicating that robots.txt contains a directive specifically permitting the crawler. See `ai-crawler-accessibility/v0.1`.

### Explicitly disallowed

A verdict in AI crawler accessibility indicating that robots.txt contains a directive specifically denying the crawler. See `ai-crawler-accessibility/v0.1`.

### FAQ schema / FAQPage

A schema.org type for FAQ-formatted Q&A content. As of 7 May 2026, Google has deprecated FAQ rich results in SERPs. FAQ schema is removed from AEO scoring at v0.1 but retained as a structured-content contributor to GEO scoring. See `aeo-scoring/v0.1` and `geo-scoring/v0.1`.

### Featured snippet

A SERP feature where Google extracts and displays a short answer to a query directly in the results, attributed to a specific source. An AEO outcome surface. See `aeo-scoring/v0.1`.

### Freshness multiplier

A value in the range [0.30, 1.00] indicating the freshness contribution of a single page, applied per page and aggregated to per-domain. See `content-freshness/v0.1`.

### Generative engine

An AI system that produces natural-language responses to user queries, drawing on training data plus optional live retrieval, and citing sources. As of May 2026 the principal generative engines are ChatGPT, Perplexity, Claude, Gemini, Google AI Overviews. See `geo-scoring/v0.1`.

### GEO — Generative Engine Optimisation

The methodology pillar measuring inclusion as a cited or mentioned source in AI-generated responses. One of three outcome pillars. See `geo-scoring/v0.1`.

### Google Business Profile (GBP)

A customer's listing in Google's local-business database. The primary signal source for local pack inclusion. See `local-seo-signals/v0.1`.

### Intent satisfaction

The degree to which a page satisfies the user's underlying intent for a query, as evaluated by the search engine through its ranking algorithms. The primary ranking driver in 2026 Google guidance. See `seo-scoring/v0.1`.

### Knowledge panel

A SERP feature displaying entity information (organisation, person, place) in a structured side-panel. An AEO outcome surface for branded and semi-branded queries. See `aeo-scoring/v0.1`.

### Local citation

A mention of the business in a local-relevant directory. Differs from a backlink in that no hyperlink is required; consistent NAP data is the signal. See `local-seo-signals/v0.1`.

### Local-intent business

A business whose customer reach is geographically bound. Triggered by `service_reach` value in `{ regional, metro, local, service-area }` or operator override. Receives the local sub-score in the SEO pillar. See `local-seo-signals/v0.1` and `seo-scoring/v0.1`.

### Local pack

The collection of local results (typically three businesses with map markers) displayed prominently in Google search results for local-intent queries. The primary commercial outcome of local SEO. See `local-seo-signals/v0.1`.

### Low confidence

A flag applied to sub-score measurements when underlying data is too sparse for the score to be reliably computed (e.g. < 20 tracked queries for a per-platform metric). The score is computed but the flag is annotated in customer-facing display. See `aeo-scoring/v0.1`, `geo-scoring/v0.1`.

### Measurement window

A defined time period over which signal events are aggregated. Default windows vary by signal: brand mentions use 90 days, per-platform affinity uses 30 days, content freshness is computed per-page at measurement time. See respective specs.

### Mention share

The customer's share of mentions across tracked queries compared to designated competitors. "Share of voice" in industry terminology. See `geo-scoring/v0.1`.

### Methodology ref

The version-tagging field carried on every finding indicating which methodology spec(s) produced it. Format: `spec-name/version`. Dual references use `+`-separation. See `methodology-overview/v0.1`.

### Multi-schema

The breadth of distinct schema.org types deployed on a page. Pages with 3+ schema types correlate with higher AI citation likelihood. See `geo-scoring/v0.1`.

### NAP consistency

The agreement of Name, Address, Phone across all online listings of a business. Inconsistency is a negative signal. See `local-seo-signals/v0.1`.

### noai / noLLM / noimageai directives

Newer directives restricting AI training and inference use of content. The `noai` directive is the broadest; `noLLM` is specific to language models; `noimageai` restricts AI use of images. Emerging across 2024–2026; not yet formal standards. See `ai-crawler-accessibility/v0.1`.

### noai directive (verdict)

A verdict in AI crawler accessibility indicating that the site emits `noai`-family directives via meta robots or X-Robots-Tag. The highest-precedence verdict; overrides robots.txt-level decisions. See `ai-crawler-accessibility/v0.1`.

### Outcome pillar

A measurement pillar that measures a customer outcome (search ranking, answer-surface inclusion, AI citation). The three outcome pillars are SEO, AEO, GEO. ARC is not an outcome pillar; it is a measurement-vocabulary spec referenced by the outcome pillars. See `methodology-overview/v0.1`.

### Page importance weighting

The relative weight assigned to a page when aggregating per-page measurements (freshness, structured content, etc.) into per-domain measurements. Pages with higher commercial relevance carry more weight. Operator-defined within each spec.

### PAA — People Also Ask

A SERP feature displaying related questions and brief answers expandable from each question. An AEO outcome surface. See `aeo-scoring/v0.1`.

### Per-platform source affinity

The platform-specific preference of each AI engine for the customer's domain relative to the platform's baseline citation rate. Measured at v0.1 via outcome (citation rate per platform) rather than proxy (Wikipedia/Reddit/YouTube presence). See `per-platform-source-affinity/v0.1`.

### Pillar runtime sub-score

The runtime-compatibility component of a pillar's sub-score set, populated by routing ARC findings from runtimes tagged with the appropriate `relevant_outcome_pillar` characteristic. See `agent-runtime-compatibility/v0.1`.

### Pillar score

The score of one of the three outcome pillars (SEO, AEO, GEO), computed as the weighted sum of the pillar's sub-scores. The pillar scores compose to the overall MakeitSEO score by arithmetic mean. See `methodology-overview/v0.1`.

### Platform average affinity

The mean affinity score observed across all measured domains on a platform within the same measurement window. The baseline against which a customer's affinity is compared. See `per-platform-source-affinity/v0.1`.

### Quality tier

A classification of the source domain hosting a brand mention. v0.1 defines three tiers: Tier 1 (high authority), Tier 2 (mid authority), Tier 3 (low authority). See `brand-web-mentions/v0.1`.

### Rankability

The traditional SEO concept of a page's position within a candidate set once it has entered. Distinct from retrievability. See `methodology-overview/v0.1`.

### Relative affinity

The ratio of a customer's affinity to the platform average: `affinity_score / platform_avg_affinity`. A value of 1.0 means the customer is cited at the platform's average rate. See `per-platform-source-affinity/v0.1`.

### Retrievability

The property of a page being eligible to enter the candidate set that a search or AI system considers. Distinct from rankability. A derived cross-cutting view in the methodology; not a pillar score. See `methodology-adr/v0.1` Decision 15 and `methodology-overview/v0.1`.

### Review velocity

The frequency of new reviews on a customer's GBP and other review platforms. Review velocity affects local SEO outcomes alongside review volume and sentiment. See `local-seo-signals/v0.1`.

### Runtime

An execution environment that mediates between an AI model and external systems (websites, APIs, databases, files). Responsible for fetching, parsing, authenticating, managing state, and returning results to the model. See `agent-runtime-compatibility/v0.1`.

### Runtime category

A high-level grouping of runtimes by architectural pattern. ARC v0.1 defines three categories: `browser-execution`, `headless-fetch`, `model-native-harness`. See `agent-runtime-compatibility/v0.1`.

### Runtime readability

The property of a website such that an agent runtime can successfully fetch, parse, and return its content to a downstream model. A high-runtime-readability site is legible to a wide variety of runtimes. See `agent-runtime-compatibility/v0.1`.

### Schema gap

A cascade event type indicating a material reduction in the customer's structured-content coverage between consecutive audits. Often indicates a CMS template change or deployment issue. See `cascade-monitoring/v0.1`.

### SEO — Search Engine Optimisation

The methodology pillar measuring traditional search ranking on Google, Bing, and regional engines. One of three outcome pillars. See `seo-scoring/v0.1`.

### Service reach

A customer's market-reach classification (national, international, regional, metro, local, service-area). Captured during onboarding; triggers local-intent classification when applicable. See `local-seo-signals/v0.1`.

### Severity

A classification of the urgency of operator attention required for a cascade event. v0.1 defines three: `info`, `warning`, `critical`. See `cascade-monitoring/v0.1`.

### Sub-score

A component of a pillar score. Each outcome pillar has multiple sub-scores with explicit weights summing to 100%. Sub-score weights and components are defined per pillar spec.

### Surface

A search-result or AI-response feature in which a customer's content can appear (e.g. featured snippet, AIO citation, ChatGPT citation, Perplexity citation). See `aeo-scoring/v0.1`, `geo-scoring/v0.1`.

### Task profile

A canonical workload defined in the operational infrastructure for cost-comparison purposes (e.g. FAQ generation, content brief, GEO sweep query). Each profile has explicit token sizes and tier-class requirements. See `api-cost-watchdog/v0.1` (Sprint 2.17).

### Tier

Multiple senses:
1. **Customer subscription tier:** Starter, Growth, Scale. Determines tracked-query caps, platform visibility, cadence configurations.
2. **Brand mention quality tier:** Tier 1, Tier 2, Tier 3. Quality classification of mention sources.
3. **Severity grading:** info, warning, critical.

Context-disambiguated per spec.

### Tracked query

A search-or-AI-engine query within a customer's tracked-query set. Per-query measurement is the basis for citation rate, mention share, AEO surface inclusion, and per-platform affinity. See `aeo-scoring/v0.1`, `geo-scoring/v0.1`.

### Verdict

The result of evaluating a single AI crawler's permission to fetch a site. One of: `explicitly_allowed`, `explicitly_disallowed`, `default_indeterminate`, `noai_directive`. See `ai-crawler-accessibility/v0.1`.

### Weighted mention count

A computed quantity aggregating raw mention counts across quality tiers using v0.1 tier weights. Log-scaled to produce the final brand mentions score. See `brand-web-mentions/v0.1`.
